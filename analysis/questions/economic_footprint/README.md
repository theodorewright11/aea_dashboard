# Economic Footprint

**Overarching question:** What is AI's total economic footprint across the US economy — which sectors lead, what structural patterns drive exposure, and how is the footprint changing over time?

---

## File inventory

This folder contains two distinct generations of analysis. They are kept separately so the documentation record is clean.

### Legacy files (reference only — broken dataset names)
| File | Status | Notes |
|------|--------|-------|
| `run.py` | Broken — reference only | Old single-question script using pre-overhaul dataset names |
| `economic_footprint.md` | Reference only | Original single-question report from the flat analysis era |
| `dashboard.md` | Reference only | Old dashboard notes |
| `figures/` | Old figures | Generated by the old run.py |

### Active sub-questions

| Folder | Question | Key outputs |
|--------|----------|-------------|
| `sector_footprint/` | Which sectors lead on workers, wages, and % tasks affected? | Aggregate totals, major/minor/broad breakdowns, floor vs ceiling ranges |
| `skills_landscape/` | What do SKA gaps and technology category profiles look like across the economy? | SKA domain leads (AI vs human), weighted tech category exposure by sector |
| `job_structure/` | How do job zones and job outlook distribute across exposure levels? | Zone × exposure, outlook × exposure, sector-level structural correlates |
| `ai_modes/` | What's the automation/augmentation makeup — agentic vs conversational, auto-aug distribution? | Mode split by sector, auto-aug score distribution |
| `trends/` | How are workers/wages/pct changing over time across sectors and configs? | Line charts per config and sector |
| `state_profiles/` | Do states share similar AI exposure profiles? | State rankings, cluster analysis, regional patterns |
| `work_activities/` | What does the GWA/IWA/DWA footprint look like with the same lenses? | WA-level footprint, mode split, trends |

## Five dataset configs

All sub-questions use the five canonical `ANALYSIS_CONFIGS` (defined in `analysis/config.py`). Primary = `all_confirmed`.

| Key | Dataset | Story |
|-----|---------|-------|
| `all_confirmed` | `AEI Both + Micro 2026-02-12` | **PRIMARY** — confirmed usage (conv + API + Microsoft) |
| `all_ceiling` | `All 2026-02-18` | Ceiling — everything AI can reach |
| `human_conversation` | `AEI Conv + Micro 2026-02-12` | Confirmed human conversational use |
| `agentic_confirmed` | `MCP + API 2026-02-12` | Confirmed agentic tool-use |
| `agentic_ceiling` | `MCP + API 2026-02-18` | Agentic ceiling (most recent) |

## How to run

Each sub-question is independent (no cross-dependencies):

```bash
venv/Scripts/python -m analysis.questions.economic_footprint.sector_footprint.run
venv/Scripts/python -m analysis.questions.economic_footprint.skills_landscape.run
venv/Scripts/python -m analysis.questions.economic_footprint.job_structure.run
venv/Scripts/python -m analysis.questions.economic_footprint.ai_modes.run
venv/Scripts/python -m analysis.questions.economic_footprint.trends.run
venv/Scripts/python -m analysis.questions.economic_footprint.state_profiles.run
venv/Scripts/python -m analysis.questions.economic_footprint.work_activities.run
```

## Notes

- The `work_activities/` sub-folder uses `compute_work_activities()` from `backend.compute` since `get_group_data` only supports occupation-level aggregation.
- State clustering in `state_profiles/` uses k-means on a state × major-category pct matrix; k=5 is the default but can be changed at the top of `run.py`.
- See `analysis/questions/work_activity_exposure/` for deeper WA-level analysis run by a parallel agent. `work_activities/` here covers the economic footprint lens only; it does not duplicate the full WA exposure analysis.
